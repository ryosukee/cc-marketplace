# debrief

セッション終了前の棚卸し。現在の状態を全方位で確認し、draft に記録する。

## ワークフロー

### ステップ 1: 物理状態の確認

以下を実行して結果を記録する:

- `git branch --show-current`: 現在のブランチ
- `git status -s`: 未コミット変更
- `git worktree list`: worktree の有無
- `git stash list`: stash の有無
- `git log --oneline`: 本セッション起点のコミット履歴

セッション中に触れた他リポジトリがある場合、そのリポジトリでも `git status -s` を実行する。

異常な状態 (main 上の未コミット、宙に浮いた worktree、stash 残留等) があれば警告として記録する。

### ステップ 2: 完了事項の抽出

会話コンテキストからこのセッションで完了した作業を抽出する:

- 成果物 (PR 番号、コミット ID、変更ファイル等)
- 設計判断、方針決定
- 外部サービスへの変更 (デプロイ、設定変更等)

### ステップ 3: タスク管理の棚卸し

プロジェクトで使われているタスク管理を確認する。
タスク管理の種類はハードコードしない。プロジェクトの CLAUDE.md、README、
設定ファイル等からタスク管理方法を推定し、それぞれについて棚卸しする。

例:

- JIRA: チケットのステータスが実態と合っているか
- GitHub Issues: 完了した issue が close されているか
- Linear: タスクのステータス更新漏れがないか
- プロジェクト内の TODO ファイル: 完了した項目が残っていないか

タスク管理が特定できない場合はこのステップをスキップする。

### ステップ 4: 未完了タスクの洗い出し

TaskList を確認し、未完了 (in_progress, pending) のタスクを一覧化する。

会話コンテキストから、タスク化されていないが未完了の作業も洗い出す:

- やろうとしていたが着手しなかったこと
- ユーザーから要求があったが完了しなかったこと
- 議論の中で出たが実行されなかった改善案
- 途中で方針変更して取りやめたもの

### ステップ 5: draft への書き込み

`${CLAUDE_PLUGIN_ROOT}/references/handover-init.md` を Read し、記載された手順に従って `.handover/` の存在確認と初期化を行う。
特定された `.handover/` の絶対パスを以降のステップすべてで使う。

対象 draft は上記 handover-init.md の「draft の特定」に従う。
draft が存在しない場合は新規作成する (slug はセッション内容から決定)。

書き込むセクション (handover テンプレートのセクション番号に対応):

- セクション 4「現在地」: 会話コンテキストから「何をしている途中か」を抽出して記述
- セクション 5「現在の作業状態」: ステップ 1 の物理状態の結果
- セクション 6「完了したこと」: ステップ 2 の完了事項
- セクション 7「タスク管理の棚卸し結果」: ステップ 3 の結果
- セクション 8「参照すべき資料」: セッション中に参照・作成したドキュメント、設計書、外部 URL 等を収集して記載
- セクション 9「コミット履歴」: ステップ 1 の `git log` の結果
- セクション 11「やり残し・拾い残し」: ステップ 4 の未完了タスク

### ステップ 6: ユーザーへの報告

棚卸し結果のサマリをユーザーに提示する。
警告がある場合は対応要否を確認する。
