# start

前セッションの引き継ぎ資料を読み込み、コンテキストを復元し、方向を提案する。

## ワークフロー

### ステップ 1: .handover/ ディレクトリの確認

`${CLAUDE_PLUGIN_ROOT}/references/handover-init.md` を Read し、記載された手順に従って `.handover/` の存在確認と初期化を行う。
特定された `.handover/` の絶対パスを以降のステップすべてで使う。

初回作成の場合はステップ 4 に進む。

### ステップ 2: todo の読み込み

`.handover/todo/` 内の全ファイルを Read する (ステップ 1 で特定したパス基準)。

ファイルがない場合はステップ 4 に進む。

### ステップ 3: ユーザーへの提示と判断委譲

読み込んだ全 handover の内容を統合して以下を提示する:

- 各 handover の概要 (背景、現在地、完了事項)
- 未完了タスク一覧 (TaskCreate 対象 / 箇条書き参考の分類付き)
- やり残し・拾い残し
- 懸念・リスク

提示後、ユーザーに判断を委ねる:

- タスクの復元: TaskCreate 対象として記載されたタスクを復元するか
- 方向: 前回の続きをするか、別のことをするか
- handover の処理: 読み込んだ todo を archive に移動してよいか

ユーザーの指示に従って:

- 承認されたタスクを TaskCreate で登録する
- todo/ のファイルを archive/ に移動する

TaskCreate 系ツールが提供されていない環境では、登録の代わりに draft にタスクボード節を作り、
進行管理の正とする（この制約の追跡は claude-known-issues plugin の台帳エントリ
`task-tools-unavailable` が管理する）。

### ステップ 4: draft の作成

`.handover/draft/` に新しい draft ファイルを作成する。

- ファイル名: セッション内容を表す slug を仮で付ける (例: `continue-auth-refactor.md`)
- 内容: handover テンプレートの骨格 (セクション見出しのみ)
- ステップ 3 で読み込んだ情報があれば「背景」「ゴール・原則」に転記する

既に `.handover/draft/` にファイルが存在する場合 (前回 end せず終了):

- 対象 draft は上記 handover-init.md の「draft の特定」に従う
- 既存 draft の内容をユーザーに提示する
- 再利用するか新規作成するか確認する

### ステップ 5: 方向提案

復元したコンテキストに基づいて、今日やることの候補をいくつか提案する。
AskUserQuestion は使わず、テキスト出力のみ。ユーザーが自由に返答できるようにする。

## エラーハンドリング

| エラー | 解決方法 |
|--------|---------|
| .handover/ が存在しない | ステップ 1 で作成を提案 |
| todo/ が空 | ステップ 4 に進む (初回または前回 handover なし) |
| draft/ に既存ファイル | ユーザーに再利用/新規作成を確認 |
