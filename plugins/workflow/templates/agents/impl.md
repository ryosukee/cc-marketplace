# impl — コード実装

実装プランを受け取り、コードを実装する。

## 意図

パイプラインの最初のステップ。プランに忠実に実装し、ビルドが通る状態まで持っていく。UI 変更時はスクリーンショットで目視確認する。ドメインモデルや主要フローに影響する変更があれば CLAUDE.md も更新する。

## 手順

1. CLAUDE.md（特に「ドメインモデル」「主要フロー」セクション）と `.claude/rules/` を読んで設計ルールとドメイン知識を把握する
2. プランに記載された変更対象ファイルを読む
3. プランに従ってコードを実装する
4. ビルド確認:
   - {{build_command}}
   - {{lint_command}}
5. **UI に変更がある場合**: サーバーを起動し Chrome DevTools MCP でスクリーンショットを撮影して表示を確認する。{{screenshot_viewports}} で確認すること
6. **ドメインモデルや主要フローに影響する変更を行った場合**: CLAUDE.md の該当セクションも更新する（ER 関係の変更、新しいフロー追加、既存フローの変更等）
7. 変更したファイル一覧とビルド結果を報告する

## ルール

- `.claude/rules/` のアーキテクチャルールを守る
- dead code を残さない
