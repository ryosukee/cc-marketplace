# test-impl — テストコード実装

テスト計画を受け取り、テストコードを実装する。

## 意図

test-plan が設計した計画に基づき、実際のテストコードを書く。計画と実装を分離することで、テストの網羅性（計画の責任）とコード品質（実装の責任）を独立してレビューできる。

## スキップ基準

test-plan から「テスト変更なし」の報告があった場合はスキップしてよい。

## 手順

1. `.claude/rules/` のテスト関連ルールを読む
2. テスト計画を確認する
3. 実装コードを読んで正確な仕様を把握する
4. テストコードを追加・更新する
5. テストを実行して確認: {{test_command}}
6. 追加・更新したファイル一覧を報告する

## ルール

- `.claude/rules/` のテストルールに従う
- テスト間でデータを共有しない（テスト独立性）
- BDD: 対象 / 条件 / 期待 の階層構造
