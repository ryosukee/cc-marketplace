# 実装プラン作成

ユーザーと対話しながら実装プランを作成する。作成したプランは `plans/` に保存し、`/team-implement` の入力として使う。

## 意図

いきなりコードを書くのではなく、まず「何をどう変更するか」を設計する。ユーザーとの壁打ちで要件の抜け漏れを防ぎ、team-implement パイプラインが自律的に動ける精度のプランを作る。

## 引数

実装したい機能の概要（自然言語）

## ワークフロー

### 1. 要件の理解

- 引数で渡された概要を元に、関連するコードを Explore agent で調査する
- 不明点があれば AskUserQuestion で確認する

### 2. プランのドラフト

以下の構成でプランを作成する:

```markdown
# {機能名}

## 概要
{何をなぜ作るか}

## 変更対象ファイル
| ファイル | 変更内容 |
|---|---|

## 実装詳細
{各ファイルの変更内容を具体的に記述}

## テスト方針
{どのテストグループに追加するか、新規 spec が必要か}
```

### 3. ユーザーとの壁打ち

- ドラフトを提示し、フィードバックをもらう
- 設計判断が必要な箇所は AskUserQuestion で確認
- 修正を繰り返してプランを確定させる

### 4. プランファイルの保存

- `plans/{名前}.md` に保存
- ユーザーに「`/team-implement` でこのプランを実行できます」と案内

## 注意

- プラン作成段階ではコードを変更しない
- 既存のアーキテクチャルール (`.claude/rules/`) を参照して設計する
