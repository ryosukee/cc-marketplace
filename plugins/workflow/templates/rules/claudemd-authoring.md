# CLAUDE.md 編集ルール

## 役割

CLAUDE.md はセッション開始時にコンテキストに読み込まれるプロジェクトガイド。コードの書き方ではなく、プロジェクトの構造・ビルド方法・ドメイン知識を伝える。

## サイズ

200 行以下を目標にする。超える場合は `.claude/rules/` に分離するか `@path` で import する。長いほどコンテキストを消費し、指示の遵守率が下がる。

## 書くべきもの

- ビルド・起動コマンド
- 検証コマンド (lint, test)
- ディレクトリ構成の概要
- ドメインモデル (ER 関係) と主要フロー
- 環境構成 (開発/本番/CI の違い)
- 既知の制約・注意事項
- Worker 選定ルール等のワークフロー規約

## 書かないもの（rules に分離）

- 言語・フレームワーク固有のコーディングルール → rules ファイルへ
- テスト規約の詳細 → rules ファイルへ
- コンポーネント設計の詳細 → rules ファイルへ

## 重複の禁止

rules ファイルに詳細がある場合、CLAUDE.md には概要 1〜2 行 + 「詳細は `.claude/rules/xxx.md` を参照」のリンクのみ書く。同じ内容を両方に書くと陳腐化する。

## ドメインモデル・主要フローの更新

機能追加・DB スキーマ変更・フロー変更時に「ドメインモデル」「主要フロー」セクションを更新する。impl agent がコード変更時に合わせて更新し、code-review / meta-review が整合性をチェックする。

## 言語

日本語で書く。セクション見出しも日本語。
