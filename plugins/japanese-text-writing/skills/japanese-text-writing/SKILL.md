# 日本語テキスト執筆規範

日本語テキストの執筆・推敲の規範。全種類に適用する共通原則（この文書）と、
テキスト種類ごとの個別規範（references/ の 5 ファイル）で構成する。

## 使い方

1. 書く（推敲する）テキストの種類を下の判定表で決める
2. 共通原則を適用する
3. 該当する分類の reference を読み、個別規範を重ねる

- ジャンルが混在する文書は節単位で分類を切り替える
  （例: README の中の設計判断の節には判断文書の規範を適用する）
- 境界事例は主目的で判定する: 再現させたい → 参照、判断させたい → 判断、
  学術的に主張・証明したい → 論文、理解させたい → 解説、引き込みたい → 読み物

## 共通原則

### 構成: 結論から書く（参照・判断・論文）

- 実用文書（参照・判断・論文）では、文書・節・段落のどれも読者が最初に知りたい答えから始める（重点先行）。
  背景や経緯は結論の後ろに置き、冒頭に地図（目的、扱う範囲、読み方）を置く。
  論文は文書レベルではアブストラクトで結論を前置きし、本文の節順は定型構成（IMRaD 等）に従う
- 解説・読み物では、構成の順序（概念の導入順・情報の出しどころ）を分類別規範に従って設計する。
  構成順について共通原則と分類別規範が衝突したら、分類別規範を優先する
- 一つの段落には一つのトピックだけを置く。段落の最初の文で、その段落が何の話か分かるようにする（全分類共通）

### 冗長の排除

- 一つの主張は一度だけ書く。言い換えの繰り返し、直後の要約し直し、隣接する節の役割重複を除く
- 削るかどうかの基準: その文が新しく伝えるのは対象のことか、文章自身のことか。
  文章自身しか語らない文（進行の実況・見え方の解説・執筆予定の宣言）は削る。
  削ると論理が飛ぶ場合は、対象側の内容を語る文に書き換える
- 読者が自力で補える中間段階の説明は書かない
- 数文にわたる議論を一文に圧縮できるなら、圧縮した一文だけを残す

### 省略の禁止

- 短さのために重要な情報を落とさない。削ってよいのは冗長（同じ情報の重複）だけで、
  前提・条件・例外・反例・数値は圧縮後も残す
- 助詞を省略しない。「は・を・に・から」で語と語の関係を明示する
- 列挙を「など」でぼかして情報を落とさない。網羅できるなら全件書き、省くなら省いたことを明示する

### 事実・意見・不確実性

- 事実と意見（解釈・評価）を文単位で区別する。意見は意見と分かる形で書く
- 数値・性能・実績には「実測」か「推定（未実測）」かを付ける
- 根拠が本文内で確定していることは断定する。不確実なことは不確実なまま書き、
  ヘッジ（「〜の可能性がある」「〜かもしれない」）を保持する。
  削ってよいのは根拠なく重ねた緩和（同一文に複数のヘッジ、文書全体での高頻度）だけ
- 因果を主張するときは機構（なぜそうなるか）を一文添える
- 検出・保証・解決を「必ず」できるかのように書かない。条件付きで正確に述べる

### 読み手の負荷

- 用語は 1 概念 1 語。同義語をローテーションしない。
  新しい概念に名前を付ける前に既存の一般用語を探し、独自語を作らない
- 専門用語・略語は初出で定義する
- 後で参照する必要のない固有名（ファイル名・関数名・識別子）を本文に出さず、一般的な言い方で済ませる
- 修飾語は係り先の直前に置く。複合語句のスコープが曖昧なら「」で括る
- 一文を長くしすぎない。接続の多い長文は文を分ける。
  ただし導入で必要な文脈共有（範囲・観点・前提）を削って短くしない
- 二重否定を使わない。同じ助詞の連続（「の」の三連など）を避ける

### AI 口調の禁止

LLM が出しがちな、情報を足さずに「書いている感」だけを付ける型。使わない。

- 誇張形容: 「非常に重要」「画期的」「革新的」「強力な」「シームレスな」。
  根拠のない評価語を消し、事実と数値で示す
- 空虚な予告と総括: 「本章では〜を見ていく」「まとめると」「要するに」（直前の言い換えだけのとき）。
  実用文書の冒頭の地図（目的・範囲の明示）は予告ではなく構成要素なので書く
- 分析風の付け足し: 「〜を浮き彫りにしている」「〜を示唆している」。事実で文を止める
- 曖昧な権威付け: 「専門家によると」「一般的に〜とされている」。出典を示すか、文ごと削る
- 装飾の型: `**ラベル**:` で始まる箇条書きの連発、絵文字 bullet、項目数を 3 に揃えた列挙
- 文頭接続詞の連発: 同一段落内で 2 文以上連続の文頭接続詞（「さらに」「また」「そのため」）を置かない。
  段落頭で前段落との論理関係を示す接続表現は対象外
- チャット残留: 「以下に示します」「お役に立てば幸いです」「素晴らしい質問ですね」
- 回りくどい繋辞: 「〜として位置づけられている」「〜と言えるでしょう」は「〜だ」に直すか削る
